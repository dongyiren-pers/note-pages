# 基本概念

图

图是一个三元组 ⟨V(G),E(G),ϕG⟩\langle V(G),E(G),\phi_G \rangle⟨V(G),E(G),ϕG​⟩，其中 V(G)V(G)V(G) 是一个非空的结点集合，E(G)E(G)E(G) 是边集合，ϕG\phi_GϕG​ 是从边集合 EEE 到结点集合上的函数。

简而言之就是，图由边集、点集和边和点的关联关系组成。

有向与无向
  • 无向边：若边 eie_iei​ 与结点无序偶 (vj,vk)(v_j,v_k)(vj​,vk​) 关联，则称该边为 无向边。即边所连两个结点没有顺序关系。
  • 有向边：若边 eie_iei​ 与结点有序偶 ⟨vj,vk⟩\langle v_j,v_k \rangle⟨vj​,vk​⟩ 关联，则称该边为 有向边。即边所连两个结点存在顺序关系。

偶是指一对，有序偶是指有顺序关系的一对。无序偶是指没有顺序关系的一对。无序偶用 (a,b)(a,b)(a,b) 表示，有序偶用 ⟨a,b⟩\langle a, b\rangle⟨a,b⟩ 表示。结点有序偶是指有顺序关系的两个结点。

  • 有向图：每一条边都是有向边的图称为 有向图。
  • 无向图：每一条边都是无向边的图称为 无向图。
邻接与孤立
  • 邻接点：若两个结点之间存在边相连，则这两个结点称为 邻接点。
  • 邻接边：若两条边关联同一个结点，则这两条边称为 邻接边。
  • 孤立点：不与任何结点相邻接的结点，称为 孤立结点 或 孤立点。
  • 零图：仅有孤立结点组成的图称为 零图。
  • 平凡图：仅有一个孤立结点的图称为 平凡图。
  • 稀疏图：边的条数 ∣E∣|E|∣E∣ 不超过 ∣V∣2|V|^2∣V∣2 的图称为 稀疏图（Sparse Graph），即边数很少的图。

  • 多重图：含有平行边的图称为 多重图。

  • 完全图：每一对结点间都有边相连的简单图称为 完全图。有 nnn 个结点的无向完全图记作 KnK_nKn​。

# 邻接链表

创建 VVV 个元素的数组，每个元素都是一个链表，每条链表表示与每个结点相连的所有其他结点。这种图的表示方法称为邻接链表，编程时通常使用这种方法表示图，使用 AdjAdjAdj 表示该邻接链表数组。

对于无向图而言，viv_ivi​ 的链表中记录所有 viv_ivi​ 的邻接点，因此 AdjAdjAdj 中所有邻接链表长度和为 2E2E2E。

对于有向图而言，viv_ivi​ 的链表仅记录所有 viv_ivi​ 射出的边，因此 AdjAdjAdj 中所有邻接链表长度和为 EEE。

邻接链表的性质
  • 邻接链表的空间消耗为 O(E+V)O(E + V)O(E+V)。
  • 邻接链表可以直接判断边的度或出度，但无法直接判断二结点是否邻接，必须遍历某结点的邻接链表。
  • 对于零图，数组中所有链表都为空，因此零图的邻接链表是一个所有元素都为空的数组。
  • 邻接链表非常适合稀疏图的表示。

# 图的遍历

各种图搜索之间的区别，体现为边分类结果的不同，以及所得遍历树（森林） 的结构差异。其决定因素在于，搜索过程中的每一步迭代，将依照何种策略来选取下一接受访问的顶点。

通常，都是选取某个已访问到的顶点的邻居。同一顶点所有邻居之间的优先级，在多数遍历中不必讲究。因此，实质的差异应体现在，当有多个顶点已被访问到，应该优先从谁的邻居中选取下一顶点。

# 深度优先搜索

深度优先搜索（Depth-First Search, DFS）选取下一顶点的策略，可概括为：优先选取最后一个被访问到的顶点的邻居。

以顶点 sss 为基点的 DFS，将首先访问顶点 sss；再从 sss 所有尚未访问到的邻居中任取其一，并以之为基点，递归地执行 DFS。

# 广度优先搜索

深度优先搜索（Breadth-First Search, BFS）选取下一顶点的策略，可概括为：优先选取最先一个被访问到的顶点的邻居。

以顶点 sss 为基点的 BFS，将首先访问顶点 sss；再依次访问 sss 所有尚未访问的邻居；再按后者被访问的先后次序，逐个访问它们的邻居；...； 如此不断。在有已访问到的顶点中，仍有邻居尚未访问者，构成 前沿集（frontier）。于是，BFS 过程也可等效地理解为：

反复从前沿集中找到最早被访问到顶点 vvv，若其邻居均已访问到，将其逐出前沿集；否则，随意选出一个尚未访问的邻居，并将其加入到前沿集中。

由于每一步迭代都有一个顶点被访问，故至多迭代 O(n)O(n)O(n) 步。另一方面， 因为不会遗漏每个刚被访问顶点的任何邻居，故对于无向图必能覆盖 sss 所属的连通分量 ，对于有向图必能覆盖以 sss 为起点的可达分量。倘若还有来自其它连通分量或可达分量的顶点，则不妨从该顶点出发，重复上述过程。

# 拓扑排序

对一个有向无环图 G 进行 拓扑排序（Topological Sorting）是将 G 中所有顶点排成一个线性序列，使得图中任意一对顶点 uuu 和 vvv ，若边 <u,v>∈E(G)<u,v> ∈ E(G)<u,v>∈E(G)，则 uuu 在线性序列中出现在 vvv 之前。简单的说，由某个集合上的一个偏序得到该集合上的一个全序，这个操作称之为拓扑排序。

# Kahn 算法

初始状态下，集合 AAA 装着所有入度为 000 的点，BBB 是一个空列表。

每次从 AAA 中取出一个点 uuu（可以随便取）放入 BBB，然后将 uuu 的所有边 (u,v1),(u,v2),(u,v3)⋯(u, v_1), (u, v_2), (u, v_3) \cdots(u,v1​),(u,v2​),(u,v3​)⋯ 删除。对于边 (u,v)(u, v)(u,v)，若将该边删除后点 vvv 的入度变为 000，则将 vvv 放入 AAA 中。

不断重复以上过程，直到集合 AAA 为空。检查图中是否存在任何边，如果有，那么这个图一定有环路，否则返回 BBB，BBB 中顶点的顺序就是拓扑排序的结果。

# 基于 DFS 的拓扑排序

在 DFS 完成一个节点的遍历后（即从该节点出发的所有可达节点都已访问），将该节点加入到一个栈或列表中。这一步是拓扑排序的关键，因为它确保了任何依赖当前节点的节点都已经在当前节点之前被加入到栈中。

完成对所有节点的 DFS 遍历后，栈中的元素顺序即为图的一个有效的拓扑排序。从栈顶开始弹出元素，得到的顺序即是一个拓扑排序的结果。

# Bellman-Ford 算法

算法描述

Bellman-Ford 算法是一种求解 加权图 上 单源最短路径 的算法。

Bellman-Ford 算法可以处理图中包含负权边的情况，不过不能有负权回路（即总权重为负的闭环）。

算法的基本思路是通过不断地松弛边来逐步更新到各顶点的最短路径估计。以下是算法的详细步骤：

  • 初始化：首先，将所有顶点的 disdisdis 值设置为无穷大，只有源点 sss 的 disdisdis 值设置为 0，因为它到自身的距离为 0。
  • 松弛边：然后，算法对所有的边进行 n−1n-1n−1 轮迭代。在每一轮迭代中，算法都会遍历所有的边 (u,v)(u,v)(u,v) 并更新顶点 vvv 的 disdisdis 值。如果通过边 (u,v)(u,v)(u,v) 到达 vvv 的路径比当前记录的路径更短，则更新 dis(v)dis(v)dis(v) 为 dis(u)+w(u,v)dis(u) + w(u,v)dis(u)+w(u,v)。这个步骤称为边的松弛。
  • 检测负权回路：在完成 n−1n-1n−1 轮松弛之后，算法会进行第 nnn 轮迭代来检测图中是否存在负权回路。如果在这一轮中还能松弛某条边，则说明图中存在从源点可达的负权回路，因为在一个没有负权回路的图中，最短路径最多包含 n−1n-1n−1 条边。
  • 完成：如果第 nnn 轮迭代中没有发现可以松弛的边，则算法终止。此时，dis(u)dis(u)dis(u) 对于所有顶点 uuu 来说就是从 sss 到 uuu 的最短路径长度。

# Dijkstra 算法

算法描述

Dijkstra 算法是一种求解 非负权图 上 单源最短路径 的算法。

将结点分成两个集合：已确定最短路长度的点集（记为 AAA 集合）的和未确定最短路长度的点集（记为 BBB 集合）。一开始所有的点都属于 BBB 集合。

初始化 dis(s)=0dis(s)=0dis(s)=0，其他点的 disdisdis 均为 +∞+\infty+∞。

然后重复这些操作：

  • 从 BBB 集合中，选取一个最短路长度最小的结点，移到 AAA 集合中。
  • 对那些刚刚被加入 AAA 集合的结点的所有出边执行松弛操作。

直到 BBB 集合为空，算法结束。

# 最小生成树

给定一个连通的无向图 GGG，且它的每条边均有相应的长度或权值。最小生成树（MST） 是一个包括 GGG 的所有顶点和一个边的子集的图，边的子集满足下列条件：

  • 子集中所有边的权之和为类似子集中最小的；
  • 子集中的边能保证图是连通的。

# Prim 算法

算法步骤：

  • 初始化：从图中任意选择一个顶点作为起始点，并将其加入最小生成树的集合（一个顶点集合）。
  • 选择最小边：在当前最小生成树集合的所有边界顶点（即已在树中的顶点与不在树中的顶点之间的边）中找到权重最小的边，并将这条边以及它的顶点加入到最小生成树中。
  • 重复选择：重复上述步骤，直到所有顶点都包含在最小生成树中。每一步都选择连接树与非树顶点并且权重最小的边。
  • 完成：当所有顶点都加入到最小生成树后，算法完成。

# kruskal 算法

Kruskal 算法的主要特点是按照边的权重顺序处理它们，而不是按照顶点。

算法步骤：

  • 排序所有边：首先将图中的所有边按照权重进行升序排序。
  • 初始化森林：开始时，算法考虑一个森林，其中每个顶点都是一个单独的树。
  • 选择最小边：从排序后的边列表中选择权重最小的边。如果这条边连接的两个顶点属于森林中的不同树（即添加这条边不会形成回路），则将这条边加入到最小生成树中。
  • 合并树：每次加入一条边后，相关的两棵树会合并成一棵树。
  • 重复选择：重复上述过程，直到所有顶点都包含在同一棵树中，或者所有边都已经考虑完毕。
  • 完成：当所有顶点都在同一棵树中时，这棵树就是最小生成树。
