# 基础知识

# 队列

队列是典型的 先入先出 （FIFO）数据结构。

插入（insert）操作称作入队（enqueue），新元素始终被添加在队列的末尾。 删除（delete）操作称为出队（dequeue)。 只能移除第一个元素。

队列的一个有效实现是 循环数组。具体来说，我们可以使用固定大小的数组和两个指针来指示起始位置和结束位置。

  • frontfrontfront：指向队列头部第 1个有效数据的位置；
  • rearrearrear：指向队列尾部（即最后 1 个有效数据）的 下一个位置，即下一个从队尾入队元素的位置。

为了避免 队列为空 和 队列为满 的判别条件冲突，我们有意浪费一个位置：循环数组中任何时刻一定至少有一个位置不存放有效元素。

  • 判别队列为空的条件是：front==rearfront == rearfront==rear；
  • 判别队列为满的条件是：(rear+1)modcapacity==front(rear + 1)\mod capacity == front(rear+1) modcapacity==front。可以这样理解，当 rearrearrear 循环到数组的前面，要从后面追上 frontfrontfront，还差一格的时候，判定队列为满。

capacitycapacitycapacity 表示队列的容量，包括那个留空的位置。

因为有循环的出现，要特别注意处理数组下标可能越界的情况。

  • 指针后移时，下标 +1+1+1，要取模；
  • 指针前移时，为了循环到数组的末尾，需要先加上数组的长度，然后下标 −1-1−1，再对数组长度取模。

# 验证栈序列

给定 pushed 和 popped 两个序列，每个序列中的 值不重复，只有当它们可能是在最初空栈上进行的推入 push 和弹出 pop 操作序列的结果时，返回 true；否则，返回 false 。

复杂度分析

  • 时间复杂度：O(N)O(N)O(N)，其中 NNN 是 pushed 序列和 popped 序列的长度。
  • 空间复杂度：O(N)O(N)O(N)。

# 单调队列

单调队列（monotonous queue）是指队列的内部从队列头部到队列尾部满足单调性的队列结构。

以单调递减的队列为例。向单调队列内插入元素时，可能需要移除元素以保证队列内的单调性。

  • 如果要插入的元素 x 小于队列尾部元素，则 x 小于队列内任何一个元素，直接插入即可。
  • 如果要插入的元素 x 大于队列尾部元素，则需要移除部分元素，以确保插入该元素后队列的单调性。

# 单调栈

单调栈（monotonous stack）是指栈的内部从栈底到栈顶满足单调性的栈结构。

以单调递减的栈为例。向单调栈内插入元素时，可能需要弹出元素以保证栈内的单调性。

  • 如果要插入的元素 x 小于栈顶元素，则 x 小于栈内任何一个元素，直接插入即可。
  • 如果要插入的元素 x 大于栈顶元素，则需要弹出部分元素，以确保插入该元素后栈的单调性。
