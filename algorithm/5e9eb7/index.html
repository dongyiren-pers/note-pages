# 动态规划

# 基础知识

动态规划（英语：Dynamic programming，简称 DP）是一种在数学、管理科学、计算机科学、经济学和生物信息学中使用的，通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。

动态规划不是某一种具体的算法，而是一种算法思想：若要解一个给定问题，我们需要解其不同部分（即子问题），再根据子问题的解以得出原问题的解。

动态规划常常适用于有重叠子问题和最优子结构性质的问题，并且记录所有子问题的结果，因此动态规划方法所耗时间往往远少于朴素解法。

动态规划有自底向上和自顶向下两种解决问题的方式。自顶向下即记忆化递归，自底向上就是递推。

使用动态规划解决的问题有一个明显的特点，一旦一个子问题的求解得到结果，以后的计算过程就不会再修改它，这样的特点叫做无后效性，求解问题的过程形成了一张有向无环图。动态规划只解决每个子问题一次，具有天然剪枝的功能，从而减少计算量。

# 最优子结构

最优子结构规定的是子问题与原问题的关系。

动态规划要解决的都是一些问题的最优解，即从很多解决问题的方案中找到最优的一个。当我们在求一个问题最优解的时候，如果可以把这个问题分解成多个子问题，然后递归地找到每个子问题的最优解，最后通过一定的数学方法对各个子问题的最优解进行组合得出最终的结果。总结来说就是一个问题的最优解是由它的各个子问题的最优解决定的。

# 重叠子问题

重叠子问题规定的是子问题与子问题的关系。

当我们在递归地寻找每个子问题的最优解时，有可能会重复地遇到一些更小的子问题，而且这些子问题会重叠地出现在子问题里，出现这样的情况，会有很多重复的计算，动态规划可以保证每个重叠的子问题只被求解一次。当重复的问题很多的时候，动态规划可以减少很多重复的计算。

重叠子问题不是保证解的正确性必须的，但是如果递归求解子问题时，没有出现重叠子问题，则没有必要使用动态规划，直接普通的递归就可以了。

# 解决问题的核心

解决动态规划问题的核心：找出子问题及其子问题与原问题的关系。

找到了子问题以及子问题与原问题的关系，就可以递归地求解子问题了。但重叠子问题使得直接递归会有很多重复计算，于是就想到记忆化递归法：若能事先确定子问题的范围就可以建表存储子问题的答案。

# NP 完全问题

# P/NP/NPC 问题

NP 的英文全称是 Non-deterministic Polynomial 的问题，即多项式复杂程度的非确定性问题。简单的写法是 NP=P?NP=P?NP=P?。

P 类问题：一类可以在多项式时间内求解的判定问题。

判定问题：判断是否有一种能够解决某一类问题的能行算法的研究课题。

NP 类问题：一类可以在非确定性多项式时间内求解的判定问题。

非确定性算法：非确定性算法将问题分解成猜测和验证两个阶段。算法的猜测阶段是非确定性的，算法的验证阶段是确定性的，它验证猜测阶段给出解的正确性。设算法 A 是解一个判定问题 Q 的非确定性算法，如果 A 的验证阶段能在多项式时间内完成，则称 A 是一个多项式时间非确定性算法。有些计算问题是确定性的，例如加减乘除，只要按照公式推导，按部就班一步步来，就可以得到结果。但是，有些问题是无法按部就班直接地计算出来。比如，找大质数的问题。有没有一个公式能推出下一个质数是多少呢？这种问题的答案，是无法直接计算得到的，只能通过间接的 “猜算” 来得到结果。这也就是非确定性问题。而这些问题的通常有个算法，它不能直接告诉你答案是什么，但可以告诉你，某个可能的结果是正确的答案还是错误的。这个可以告诉你 “猜算” 的答案正确与否的算法，假如可以在多项式（polynomial）时间内算出来，就叫做多项式非确定性问题。

NPC 问题：NP 中的某些问题的复杂性与整个类的复杂性相关联。这些问题中任何一个存在多项式时间的算法，那么所有 NP 问题都是多项式时间可解的。这些问题被称为 NP-完全问题（NPC 问题）。

# 背包 DP

背包问题（Knapsack problem）是一种组合优化的 NP 完全问题。背包问题可以描述为：给定一组物品，每种物品都有自己的重量和价格，在限定的总重量内，我们如何选择，才能使得物品的总价格最高。

# 0−1 背包问题

最基本的背包问题就是 0-1 背包问题：一共有 NNN 件物品，第 iii（iii 从 111 开始）件物品的重量为 w[i]w[i]w[i]，价值为 v[i]v[i]v[i]。总重量不超过背包承载上限 WWW，能够装入背包的最大价值是多少？

我们的目标是背包内物品的总价值，而变量是物品和书包的限重，可定义状态 dpdpdp： dp[i][j]dp[i][j]dp[i][j] 表示将前 iii 件物品装进限重为 jjj 的背包可以获得的最大价值，0≤i≤N,0≤j≤W0 \le i \le N, 0 \le j \le W0≤i≤N,0≤j≤W。

将 dp[0][0...W]dp[0][0...W]dp[0][0...W] 初始化为 000，表示将前 000 个物品（即没有物品）装入书包的最大价值为 000。当 i>0i > 0i>0 时 dp[i][j]dp[i][j]dp[i][j] 有两种情况：

  • 不装入第 iii 件物品，即 dp[i−1][j]dp[i - 1][j]dp[i−1][j]；
  • 装入第 iii 件物品（前提是能装下），即 dp[i−1][j−w[i]]+v[i]dp[i - 1][j - w[i]] + v[i]dp[i−1][j−w[i]]+v[i]。

因此，状态转移方程为：dp[i][j]=max(dp[i−1][j],dp[i−1][j−w[i]]+v[i]),j>=w[i]dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - w[i]] + v[i]),j >= w[i]dp[i][j]=max(dp[i−1][j],dp[i−1][j−w[i]]+v[i]),j>=w[i]

由上述状态转移方程可知，dp[i][j]dp[i][j]dp[i][j] 的值只与 dp[i−1][0,...,j−1]dp[i-1][0,...,j-1]dp[i−1][0,...,j−1] 有关，因此可以采用 滚动数组 进行空间优化（即去掉 dp 的第一维）。需要注意的是，为了防止上一层循环的 dp[0,...,j−1]dp[0,...,j-1]dp[0,...,j−1] 被覆盖，循环时 jjj 只能逆向枚举（空间优化前没有这个限制）。

伪代码可表示为：

dp[0,...,W] = 0
for i = 1,...,N
    for j = W,...,w[i]  // 逆向枚举
        dp[j] = max(dp[j], dp[j - w[i]] + v[i])

动态规划避免重复计算的思想在 0-1 背包问题中体现得淋漓尽致。第 iii 件物品装入或不装入而获得的最大价值完全可以由前面 i−1i-1i−1 件物品的最大价值决定，暴力枚举忽略了这个事实。

# 完全背包问题

完全背包与 0-1 背包不同就是每种物品可以有无限多个： 一共有 NNN 种物品，每种物品有无限多个，第 iii（ iii 从 111 开始）种物品的重量为 w[i]w[i]w[i]，价值为 v[i]v[i]v[i]。总重量不超过背包承载上限 WWW，能够装入背包的最大价值是多少？

目标和变量和 0-1 背包没有区别，可定义与 0-1 背包问题几乎完全相同的状态 dpdpdp：dp[i][j]dp[i][j]dp[i][j] 表示将前 iii 种物品装进限重为 jjj 的背包可以获得的最大价值, 0≤i≤N,0≤j≤W0 \le i \le N, 0 \le j \le W0≤i≤N,0≤j≤W。

初始状态是一样的，我们将 dp[0][0...W]dp[0][0...W]dp[0][0...W] 初始化为 000，表示将前 000 种物品（即没有物品）装入书包的最大价值为 000。那么当 i>0i > 0i>0 时 dp[i][j]dp[i][j]dp[i][j] 有两种情况：

  • 不装入第 iii 种物品，即 dp[i−1][j]dp[i - 1][j]dp[i−1][j]，同 0-1 背包问题；
  • 装入第 iii 种物品，此时要转移到 dp[i][j−w[i]]+v[i]dp[i][j - w[i]] + v[i]dp[i][j−w[i]]+v[i]。

证明：参考 0-1 背包问题思想，完全背包问题的状态转移方程可写为（记 w=w[i],v=v[i]w = w[i],v = v[i]w=w[i],v=v[i]）：

dp[i][j]=max⁡(dp[i−1][j],dp[i][j−w]+v) dp[i][j]=\max(dp[i-1][j],dp[i][j-w]+v) dp[i][j]=max(dp[i−1][j],dp[i][j−w]+v)

因此，状态转移方程为：dp[i][j]=max⁡(dp[i−1][j],dp[i][j−w[i]]+v[i]),j≥w[i]dp[i][j] = \max(dp[i - 1][j], dp[i][j - w[i]] + v[i]), j \ge w[i]dp[i][j]=max(dp[i−1][j],dp[i][j−w[i]]+v[i]),j≥w[i]

和 0-1 背包问题类似，同样可以进行空间优化，优化后不同点在于这里的 jjj 只能正向枚举而 0-1 背包只能逆向枚举，因为这里的 maxmaxmax 第二项是 dp[i]dp[i]dp[i] 而 0-1 背包是 dp[i−1]dp[i - 1]dp[i−1]，即这里就是需要覆盖而 0-1 背包需要避免覆盖。伪代码可表示为：

for i = 1,...,N
    for j = w[i],...,W  // 正向枚举
        dp[j] = max(dp[j], dp[j - w[i]] + v[i])

# 多重背包问题

多重背包与前述问题的不同是每种物品是有限个：一共有 NNN 种物品，第 iii（iii 从 111 开始）种物品的数量为 n[i]n[i]n[i]，重量为 w[i]w[i]w[i]，价值为 v[i]v[i]v[i]。总重量不超过背包承载上限 WWW，能够装入背包的最大价值是多少？

状态转移方程为：dp[i][j]=max(dp[i−1][j−k∗w[i]]+k∗v[i])foreverykdp[i][j] = max{(dp[i-1][j - k*w[i]] + k*v[i])\ for\ every\ k}dp[i][j]=max(dp[i−1][j−k∗w[i]]+k∗v[i]) for every k

同样可以进行空间优化，而且 jjj 必须逆向枚举，优化后伪代码为

dp[0,...,W] = 0
for i = 1,...,N
    for j = W,...,w[i] //逆向枚举
        for k = [0, 1,..., min(n[i], j / w[i])]
            dp[j] = max(dp[j], dp[j - k * w[i]] + k * v[i])

# 背包问题变种

恰好装满

背包问题有时候还有一个限制是必须 恰好装满背包，此时基本思路没有区别，只是在初始化的时候有所不同。

如果没有恰好装满背包的限制，我们将 dp 全部初始化成 000 就可以了。因为任何容量的背包都有一个合法解 “什么都不装”，这个解的价值为 000。如果有恰好装满的限制，那只应该将 dp[0,...,N][0]dp[0,...,N][0]dp[0,...,N][0] 初始为 000，其它 dp 值均初始化为 −∞-\infty−∞，因为此时只有容量为 000 的背包可以在什么也不装情况下被 “恰好装满”，其它容量的背包初始均没有合法的解，应该被初始化为 −∞-\infty−∞。

求方案总数

除了给定每个物品价值求可得到的最大价值外，还有一类问题是 问装满背包或将背包装至某一指定容量的方案总数。对于这类问题，需要将状态转移方程中的 maxmaxmax 改成 sumsumsum，大体思路是不变的。例如若每件物品均是完全背包中的物品，转移方程即为 dp[i][j]=sum(dp[i−1][j],dp[i][j−w[i]]),j≥w[i]dp[i][j] = sum(dp[i-1][j], dp[i][j-w[i]]),\ j \ge w[i]dp[i][j]=sum(dp[i−1][j],dp[i][j−w[i]]), j≥w[i]。

组合问题先遍物品，排列问题先遍历背包容量。

二维背包

前面讨论的背包容量都是一个量：重量。二维背包问题是指每个背包有两个限制条件（比如重量和体积限制），选择物品必须要满足这两个条件。此类问题的解法和一维背包问题不同就是 dp 数组要多开一维，其他和一维背包完全一样。

求最优方案

一般而言，背包问题是要求一个最优值，但如果要求输出这个最优值的方案，可以参照一般动态规划问题输出方案的方法：记录下每个状态的最优值是由哪一个策略推出来的，这样便可根据这条策略找到上一个状态，从上一个状态接着向前推即可。

以 0-1 背包为例，我们可以再使用一个数组 G[i][j]G[i][j]G[i][j] 来记录方案，设 G[i][j]=0G[i][j] = 0G[i][j]=0 表示计算 dp[i][j]dp[i][j]dp[i][j] 的值时是采用了 maxmaxmax 中的前一项（即 dp[i−1][j]dp[i-1][j]dp[i−1][j]），G[i][j]=1G[i][j] = 1G[i][j]=1 表示采用了 maxmaxmax 中的后一项。即分别表示了两种策略：未装入第 iii 个物品及装了第 iii 个物品。

# 数位 DP

数位 是指把一个数字按照个、十、百、千等等一位一位地拆开，关注它每一位上的数字。如果拆的是十进制数，那么每一位数字都是 0~9，其他进制可类比十进制。

数位 DP 是来解决一类特定问题，这种问题比较好辨认，一般具有这几个特征：

  • 要求统计满足一定条件的数的数量（即，最终目的为计数）；
  • 这些条件经过转化后可以使用 数位 的思想去理解和判断；
  • 输入会提供一个数字区间（有时也只提供上界）来作为统计的限制；
  • 上界很大（比如 101810^{18}1018），暴力枚举验证会超时。

数位 DP 的基本思想：考虑人类计数的方式，最朴素的计数就是从小到大开始依次加一。但我们发现对于位数比较多的数，这样的过程中有许多重复的部分。例如，从 700070007000 数到 799979997999、从 800080008000 数到 899989998999、和从 900090009000 数到 999999999999 的过程非常相似，它们都是后三位从 000000000 变到 999999999，不一样的地方只有千位这一位，所以我们可以把这些过程归并起来，将这些过程中产生的计数答案也都存在一个 dpdpdp 数组里。此数组根据题目具体要求设置状态，用递推或 DP 的方式进行状态转移。

数位 DP 中通常会利用常规计数问题技巧，比如把一个区间内的答案拆成两部分相减：

ans[l,r]=ans[0,r]−ans[0,l−1] ans_{[l, r]} = ans_{[0, r]}-ans_{[0, l - 1]} ans[l,r]​=ans[0,r]​−ans[0,l−1]​

有了 dpdpdp 数组，接下来统计答案。统计答案可以选择记忆化搜索或迭代递推。为了不重不漏地统计所有不超过上限的答案，要从高到低枚举每一位，再考虑每一位可以填哪些数字，最后利用 dpdpdp 数组统计答案。

# 数字计数

问题：给定两个正整数 a,ba,ba,b，求在 [a,b][a,b][a,b] 中的所有整数中，每个数码（0∼90\sim90∼9）各出现了多少次。

不难发现，对于满 i\mathit{i}i 位的数，所有数字出现的次数都是相同的。设数组 dp[i]dp[i]dp[i] 为满 iii 位的数中每个数字出现的次数，此时暂时不处理前导零。则有 dp[i]=10×dp[i−1]+10i−1dp[i]=10 \times dp[i-1]+10^{i−1}dp[i]=10×dp[i−1]+10i−1，这两部分前一个是来自前 i−1i-1i−1 位数字的贡献，后一个是来自第 iii 位的数字的贡献。

有了 dp\mathit{dp}dp 数组，考虑如何统计答案。将上界按位分开，从高到低枚举，不贴着上界时，后面的位可以任意取值。贴着上界时，后面的位就只能取 000 到上界，分两部分分别计算贡献。最后考虑下前导零，第 iii 位为前导 000 时，111 到 i−1i-1i−1 位都是 000，即多算了将 i−1i-1i−1 位填满的答案，需要额外减去。

数据规模与约定

  • 对于 30%30\%30% 的数据，保证 a≤b≤106a\le b\le10^6a≤b≤106；
  • 对于 100%100\%100% 的数据，保证 1≤a≤b≤10121\le a\le b\le 10^{12}1≤a≤b≤1012。

参考代码

#include <bits/stdc++.h>
using namespace std;
const int N = 15;
typedef long long ll;
ll l, r, dp[N], sum[N], mi[N];
ll ans1[N], ans2[N];
int a[N];

inline void solve(ll n, ll *ans) {
    ll tmp = n;
    int len = 0;
    while (n) a[++len] = n % 10, n /= 10;
    for (int i = len; i >= 1; --i) {
        // 不贴着上界时, 后面的位可以任意取值
        for (int j = 0; j < 10; j++) 
            ans[j] += dp[i - 1] * a[i];
        for (int j = 0; j < a[i]; j++) 
            ans[j] += mi[i - 1];
        // 贴着上界时, 后面的位就只能取 0 到上界
        tmp -= mi[i - 1] * a[i], ans[a[i]] += tmp + 1;
        // 第 i 位为前导 0 时
        ans[0] -= mi[i - 1];
    }
}

int main() {
    scanf("%lld%lld", &l, &r);
    mi[0] = 1ll;
    // dp[i] 为满 i 位的数中每个数字出现的次数
    for (int i = 1; i <= 13; ++i) {
        // dp[i - 1] * 10 是来自前 i-1 位数字的贡献
        // mi[i - 1] 是来自第 i 位的数字的贡献
        dp[i] = dp[i - 1] * 10 + mi[i - 1];
        mi[i] = 10ll * mi[i - 1];
    }
    solve(r, ans1), solve(l - 1, ans2);
    for (int i = 0; i < 10; ++i) 
        printf("%lld ", ans1[i] - ans2[i]);
    return 0;
}
