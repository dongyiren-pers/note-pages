# 树

# 基础知识

树是一种抽象数据类型或是实现这种抽象数据类型的数据结构，用来模拟具有树状结构性质的数据集合。它是由 n(n>0)n(n>0)n(n>0) 个有限节点组成一个具有层次关系的集合。

把它叫做 树 是因为它看起来像一棵倒挂的树，也就是说它是根朝上，而叶朝下的。

树具有以下的特点：

  • 每个节点都只有有限个子节点或无子节点；
  • 没有父节点的节点称为根节点；
  • 每一个非根节点有且只有一个父节点；
  • 除了根节点外，每个子节点可以分为多个不相交的子树；
  • 树里面没有环路。

# 二叉树

定义：二叉树由结点的有限集合组成，这个集合或者为空，或者由一个 根结点 及两棵不相交的、分别称作这个根的 左子树 和 右子树 的二叉树组成。

二叉树的特殊形态

满二叉树：如果一棵二叉树的任何结点，或者是树叶，或者恰有两个非空子女的分支结点，则此二叉树称为满二叉树。（只有度为 0 或 2 的结点的二叉树）

注：国内许多教材如严蔚敏版数据结构教材中满二叉树指下面的完美二叉树。

完全二叉树：若一棵二叉树最多 只有最下面的两层结点度数可以小于 2，并且 最下面一层 的结点都 集中在该层最左边的若干位置上，则称此二叉树为完全二叉树。

完美二叉树：一个高度为 k(>=0)k(>=0)k(>=0) 且有 2k−12^k-12k−1 个结点的二叉树称为完美二叉树。

斜二叉树：完全向一边偏，形如单向链表。

扩展二叉树（扩充二叉树）：在叉树中出现空的子树（包括树叶）上增加空的树叶，使子树成为满二叉树的二叉树称之为扩充二叉树。

扩展原因：由于先序遍历序列中序遍历序列和后序遍历序列中任一个都不能确定一棵二叉树。扩展二叉树的光序遍历或后序遍历序列能唯一确定其二叉树。

# 二叉树的性质

性质 1：二叉树的第 iii 层（根为第 1 层）最多有 2(i−1)2^{(i-1)}2(i−1) 个结点。

性质 2：高度为 kkk 的二叉树至多有 2k−12^k-12k−1 个结点。

性质 3：具有 nnn 个结点的完全二叉树的高度为 ⌊log⁡2n⌋+1\lfloor \log _ { 2 } n \rfloor + 1⌊log2​n⌋+1。

性质 4：任何一棵二叉树，度为 0 的结点比度为 2 的结点多一个。

性质 5：完全二叉树某一节点编号的二进制表示与从根节点到这一节点的路径有如下关系：最高位是 1，其余各位从高到低表示从根节点到这一节点的路径，0 表示移动到左子节点，1 表示移动到右子节点。

  • 首先根元素入队
  • 当队列不为空的时候

# 前中序构造二叉树

对于任意一颗树而言，前序遍历的形式总是

[ 根节点, [左子树的前序遍历结果], [右子树的前序遍历结果] ]

即根节点总是前序遍历中的第一个节点。而中序遍历的形式总是

[ [左子树的中序遍历结果], 根节点, [右子树的中序遍历结果] ]

只要在中序遍历中定位到根节点，就可以得到左子树和右子树中的节点数目。由于同一颗子树的前序遍历和中序遍历的长度是相同的，就可以对应到前序遍历的结果中，对上述形式中的所有左右括号进行定位。

这样以来，我们得到了左子树的前序遍历和中序遍历结果，以及右子树的前序遍历和中序遍历结果，我们就可以递归地对构造出左子树和右子树，再将这两颗子树接到根节点的左右位置。

在中序遍历中对根节点进行定位时，一种简单的方法是直接扫描整个中序遍历结果并找出根节点，但这样做的时间复杂度较高。考虑使用哈希表快速定位根节点。键表示一个元素（节点的值），值表示其在中序遍历中的出现位置。在构造二叉树的过程之前，对中序遍历的列表进行一遍扫描，就可以构造出这个哈希映射。在此后构造二叉树的过程中，就只需要 O(1)O(1)O(1) 的时间对根节点进行定位了。

# 二叉搜索树

二叉搜索树：二叉搜索树或者为空，或者是满足下列条件的非空二叉树：

  • 若它的左子树非空，则左子树上所有结点的值均小于根结点的值；
  • 若它的右子树非空，则右子树上所有结点的值均大于或等于根结点的值；
  • 左右子树本身又各是一棵二叉搜索树。

二叉搜索树的性质：按照中序遍历将各结点打印出来，将得到按照由小到大的排列。

# 平衡二叉树

平衡二叉树/平衡二叉树搜索树又被称为 AVL 树，AVL 树的定义：一棵 AVL 树或者是空树，或者是具有下列性质的二叉搜索树：它的左子树和右子树都是 AVL 树，且左子树和右子树的高度之差的绝对值不超过 1。

# 哈夫曼树

哈夫曼树是一种特殊的二叉树，用于有效地编码数据。其特点在于能够最小化编码长度。以下是哈夫曼树的关键概念：

  • 结点的权：结点权值通常代表某种重要性，例如在字符编码中，它可能表示字符的出现频率。
  • 结点的带权路径长度：这是从树的根到该结点的路径长度（经过的边数）与该结点上权值的乘积。
  • 树的带权路径长度（WPL）：是树中所有叶结点的带权路径长度之和。

哈夫曼树的定义：在含有 n 个带权叶结点的二叉树中，带权路径长度最小的二叉树称为哈夫曼树，也被称为最优二叉树。

构造哈夫曼树的步骤：

  • 第 1 步：将 n 个权值 (w1，w2，…，wn) 的结点作为 n 棵只含一个结点的二叉树，构成森林 F。
  • 第 2 步：构造一个新结点，选择 F 中两个根结点权值最小的树作为左、右子树，并将新结点的权值设为左、右子树根结点的权值之和。
  • 第 3 步：从 F 中删除选出的两棵树，将新得到的树加入 F 中。
  • 第 4 步：重复步骤 2 和 3，直至 F 中只剩下一棵树。

哈夫曼树的特点：

  • 所有初始结点最终成为叶结点，权值越小的结点到根结点的路径越长。
  • 结点总数为 2n - 1。
  • 不存在度为 1 的结点。
  • 哈夫曼树并非唯一，但 WPL 是相同且最优的。

# 哈夫曼编码

哈夫曼编码 是一种广泛使用的压缩数据的方法，属于 可变长度编码。它根据字符出现的频率为每个字符分配不同长度的编码，以此来减少编码的总长度。

构造哈夫曼编码的步骤：

  • 第 1 步：字符集中的每个字符作为一个叶子结点，字符出现的频度作为结点的权值。根据上述方法构造哈夫曼树。
  • 第 2 步：从根结点到叶子结点的路径上，标记序列，0 代表转向左孩子，1 代表转向右孩子。

# 线段树

线段树可以在 O(log⁡N)O(\log N)O(logN) 的时间复杂度内实现 单点修改、区间修改、区间查询（区间求和，求区间最大值，求区间最小值）等操作。

线段树维护的信息在很多时候可以认为是满足（幺）半群的性质的信息。

一个幺半群 M=(S,∘,e)M=(S,\circ ,e)M=(S,∘,e)，其中 ∘\circ∘ 为在集合 SSS 上定义的二元运算符，幺半群具有以下性质：

  • 封闭性：∀x∈S\forall x\in S∀x∈S 和 ∀y∈S\forall y\in S∀y∈S 有 x∘y∈Sx\circ y\in Sx∘y∈S。
  • 结合律：∀x,y,z∈S\forall x,y,z\in S∀x,y,z∈S 有 (x∘y)∘z=x∘(y∘z)(x\circ y)\circ z=x\circ (y\circ z)(x∘y)∘z=x∘(y∘z)。
  • 存在幺元：即 ∃e∈S\exists e\in S∃e∈S 满足 ∀x∈S\forall x \in S∀x∈S 有 e∘x=xe\circ x=xe∘x=x，eee 为左幺元；或 x∘e=xx\circ e=xx∘e=x，eee 为右幺元。

我们观察到线段树上的信息一般满足这样的性质，一些数域上的加法与乘法自然，考虑二元的 max⁡(x,y)\max(x,y)max(x,y) 运算，此时幺元为 −∞-\infty−∞ 也满足这样的性质（一般左右幺元相同时简称为幺元）。

void build(int s, int t, int p) {
    // 对 [s,t] 区间建立线段树, 当前根的编号为 p
    if (s == t) {
        d[p] = a[s];
        return;
    }
    int m = s + ((t - s) >> 1);
    build(s, m, p * 2), build(m + 1, t, p * 2 + 1);
    // 递归对左右区间建树
    d[p] = d[p * 2] + d[(p * 2) + 1];
}

# 区间查询

区间查询，比如求区间 [l,r][l,r][l,r] 的总和（即 al+al+1+⋯+ara_l+a_{l+1}+ \cdots +a_ral​+al+1​+⋯+ar​）、求区间最大值/最小值等操作。

仍然以最开始的图为例，如果要查询区间 [1,5][1,5][1,5] 的和，则直接获取 d1d_1d1​ 的值（606060）即可。

如果要查询的区间为 [3,5][3,5][3,5]，此时就不能直接获取区间的值，但是 [3,5][3,5][3,5] 可以拆成 [3,3][3,3][3,3] 和 [4,5][4,5][4,5]，可以通过合并这两个区间的答案来求得这个区间的答案。

一般地，如果要查询的区间是 [l,r][l,r][l,r]，则可以将其拆成最多为 O(log⁡n)O(\log n)O(logn) 个 极大 的区间，合并这些区间即可求出 [l,r][l,r][l,r] 的答案。

// [l, r] 为查询区间, [s, t] 为当前节点包含的区间, p 为当前节点的编号
int getSum(int l, int r, int s, int t, int p) {
    // 当前区间为询问区间的子集时直接返回当前区间的和
    if (l <= s && t <= r)
        return d[p];
    int m = s + ((t - s) >> 1), sum = 0;
    // 如果左子节点代表的区间 [s, m] 与询问区间有交集, 则递归查询左子节点
    if (l <= m) sum += getsum(l, r, s, m, p * 2);
    // 如果右子节点代表的区间 [m + 1, t] 与询问区间有交集, 则递归查询子节点
    if (r > m) sum += getsum(l, r, m + 1, t, p * 2 + 1);
    return sum;
}

# 区间修改与懒惰标记

如果要求修改区间 [l,r][l,r][l,r]，把所有包含在区间 [l,r][l,r][l,r] 中的节点都遍历修改一次，时间复杂度无法承受。这里引入 懒惰标记。

懒惰标记，简单来说，就是通过延迟对节点信息的更改，从而减少可能不必要的操作次数。每次执行修改时，我们通过打标记的方法表明该节点对应的区间在某一次操作中被更改，但不更新该节点的子节点的信息。实质性的修改则在下一次访问带有标记的节点时才进行。

仍然以最开始的图为例，我们将执行若干次给区间内的数加上一个值的操作。我们现在给每个节点增加一个 tit_iti​，表示该节点带的标记值。

区间修改（区间加上某个值）

// [l, r] 为修改区间, c 为被修改的元素的变化量, [s, t] 为当前节点包含的区间, p 为当前节点的编号
void update(int l, int r, int c, int s, int t, int p) {
    // 当前区间为修改区间的子集时直接修改当前节点的值, 然后打标记, 结束修改
    if (l <= s && t <= r) {
        d[p] += (t - s + 1) * c, b[p] += c;
        return;
    }  
    int m = s + ((t - s) >> 1);
    // 如果当前节点的懒标记非空, 则更新当前节点两个子节点的值和懒标记值
    if (b[p] && s != t) {
        d[p * 2] += b[p] * (m - s + 1), d[p * 2 + 1] += b[p] * (t - m);
        b[p * 2] += b[p], b[p * 2 + 1] += b[p];  // 将标记下传给子节点
        b[p] = 0;                                // 清空当前节点的标记
    }
    if (l <= m) update(l, r, c, s, m, p * 2);
    if (r > m) update(l, r, c, m + 1, t, p * 2 + 1);
    d[p] = d[p * 2] + d[p * 2 + 1];
}
