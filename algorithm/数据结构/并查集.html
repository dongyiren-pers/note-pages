# 并查集

# 基础知识

我们以一个直观的问题引入 并查集（Union-Find Set）的概念。并查集又称 不相交集（Disjoint Set）。

亲戚问题： 有一群人，他们属于不同家族，同一个家族里的人互为亲戚，不同家族的人不是亲戚。已知每个人都知道自己与其他人是否有亲戚关系，求问有几个家族。

亲戚问题代表着一大类能用并查集解决的确定 连通分量 的问题。

从图论的角度来说，同一个集合内的元素是相互 连通 的，那么一个集合就是一个 连通分量 。问题涉及的元素可划归到互相 没有交集 的集合，因此又称这样的结构为 不相交集 。

并查集（不相交集）是一种描述不相交集合的数据结构，即若一个问题涉及多个元素，它们可划归到不同集合，同属一个集合内的元素等价（即可用任意一个元素作为代表，比如上述的互为亲戚即互相等价），不同集合内的元素不等价。

初始时，问题涉及的元素总是自己构成一个单元素集合，求解问题需要通过合并操作将等价元素归入一个集合中。为了能够合并等价元素，必须查找希望合并的对象元素属于哪个集合，以决定是否要执行合并。因此主要操作就是 “查找” 与 “合并”。

不相交 描述的是问题元素构成集合之后各个集合不相交的状态，并查 描述的是处理问题时的操作。

# 处理过程

# 问题

有 nnn 个节点，其中一些彼此相连，另一些没有相连。如果节点 a 与节点 b 直接相连，且节点 b 与节点 c 直接相连，那么节点 a 与节点 c 间接相连。

集合 是一组直接或间接相连的节点，组内不含其他没有相连的节点。

给定一个 n×nn \times nn×n 的矩阵 isConnectedisConnectedisConnected ，其中 isConnected[i][j]=1isConnected[i][j] = 1isConnected[i][j]=1 表示第 iii 个节点和第 jjj 个节点直接相连，而 isConnected[i][j]=0isConnected[i][j] = 0isConnected[i][j]=0 表示二者不直接相连。

返回矩阵中 集合 的数量。

# 初始化

初始时，每个节点是否与其他节点同属一集合是未知的，此时每个节点构成单元素集合。

# 合并

合并的依据是查找，假设已经实现了查找方法 find(x)find(x)find(x)，该方法返回 xxx 的当前代表。合并 xxx 和 yyy 的前提是 find(x)!=find(y)find(x) != find(y)find(x)!=find(y)，因为若 find(x)==find(y)find(x) == find(y)find(x)==find(y)，说明二者已经在同一集合中了。

具体做法如下，对于 x,yx,yx,y 两个节点，若 find(x)!=find(y)find(x) != find(y)find(x)!=find(y)，就令 parent[find(y)]=find(x)parent[find(y)] = find(x)parent[find(y)]=find(x)，当然可以是 parent[find(x)]=find(y)parent[find(x)] = find(y)parent[find(x)]=find(y)。你一开始可能想要写成 parent[x]=yparent[x] = yparent[x]=y，但马上发现这么写只能表示 yyy 是 xxx 的代表，而我们的目的是使 yyy 目前的代表 (find(y)find(y)find(y) 的返回值) 作为 xxx 的代表 (find(x)find(x)find(x) 的返回值) 的代表，这样才能使得 xxx 目前的集合整体并入 yyy 所在的集合。

// 直接求并
public void union(int x, int y) {
    int xRoot = find(x), yRoot = find(y);
    if (xRoot != yRoot) {
        parent[yRoot] = xRoot;
    }
}

在 main 方法中遍历矩阵，调用 union 执行合并的过程的写法如下：

for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        if (isConnected[i][j] == 1) uf.union(i, j);
    }
}

# 查找

初始化单元素集合时，parent[i]=iparent[i] = iparent[i]=i，如果此时合并 iii 和 jjj，令 parent[find(j)]=find(i)parent[find(j)] = find(i)parent[find(j)]=find(i)，因为刚开始 find(j)=jfind(j) = jfind(j)=j，find(i)=ifind(i) = ifind(i)=i，所以这个合并其实就是 parent[j]=iparent[j] = iparent[j]=i，并且要令此后 find(j)=ifind(j) = ifind(j)=i。我们发现，代表元的特点是 find(i)=ifind(i) = ifind(i)=i，即自己指向自己，而非代表元 find(j)=ifind(j) = ifind(j)=i (iii 是 jjj 的代表) 不具备这个特点。当 parent[x]=xparent[x] = xparent[x]=x 时，找到代表元，否则就找当前代表元的代表元，因为从 xxx 到它的代表元，一定是串联在一条链上的，而对于尾结点 ttt ，一定满足 parent[t]=tparent[t] = tparent[t]=t，因此可以保证找到 xxx 的代表元 ttt 。

// 直接查找
public int find(int x) {
    if (parent[x] == x) return x;
    return find(parent[x]);
}

unionunionunion 和 findfindfind 无论从理解上还是从实现上，都异常简单。这正是并查集的一大特色，短小精悍，以少量的代码完成高效的处理。

最终不相交集的数量可以在合并完成后对所有元素执行一次 find(x)find(x)find(x)，统计不同结果的个数得到。更聪明的做法是设置一个 unionCount=0unionCount = 0unionCount=0，表示合并的次数，在 unionunionunion 方法内添加一行代码，使得发生合并时 unionCount++unionCount++unionCount++ 实现累计，元素总数减去合并次数即为不相交集数量。

# 求并优化

若元素较多时，我们发现将得到一条很长的链，对头节点查找其代表元时，需要遍历整条链。

设想，如果能将树的高度降低，例如除根节点外的所有节点直接挂在根节点下（放射状），那么 findfindfind 的复杂度将为常数级，unionunionunion 也会因此变为常数级操作。基于这个思考，我们将提出更好的求并方法，使得两棵树合并后得到的新树拥有更低的树高。

# 按大小求并

较高的树高将导致较高的查找（及合并）复杂度。我们希望以某种策略使合并后得到树高较小的树。几乎不费思忖，得到一个很自然的想法：在合并时，不再默认将 yyy 所在树（的根）挂到 xxx 所在树（的根）上，而是先比较这两棵树的大小，让较小的树挂到较大的树上，因为较小的树的树高总是倾向于较低 。如果较小的那棵树低于较大的那棵树，合并后树高不变。

public void union(int x, int y) {
    int xRoot = find(x), yRoot = find(y);
    if (xRoot != yRoot) {
        if (size[yRoot] <= size[xRoot]) {
            parent[yRoot] = xRoot;
            size[xRoot] += size[yRoot];
        } else {
            parent[xRoot] = yRoot;
            size[yRoot] += size[xRoot];
        }
    }
}

我们新增了一个与 parentparentparent 大小相同的 sizesizesize 数组，size[i]size[i]size[i] 表示 iii 结点所在树的大小。UnionFindUnionFindUnionFind 类中需要添加 sizesizesize 数组字段和相应的初始化内容（单结点集合的大小为 111）。

# 按秩求并

当并查集的查找方法 不具有带路径压缩 的效果时，本节所述方法就是严格的 按高度求并。

当并查集应用了 带路径压缩 的查找方法时，heightheightheight 将不能表示严格的 树高 的概念，为严谨，需改称 按高度求并 为 按秩求并。

类似 按大小求并 的做法，我们可以立即写出 按秩 (高度) 求并 的如下方法（初始化 rankrankrank 数组写法与 sizesizesize 数组一致，此处省略）。

public void union(int x, int y) {
    int xRoot = find(x), yRoot = find(y);
    if (xRoot != yRoot) {
        if (rank[yRoot] <= rank[xRoot]) parent[yRoot] = xRoot;
        else parent[xRoot] = yRoot;
        if (rank[xRoot] == rank[yRoot]) rank[xRoot]++;
    }
}

当两棵树秩（高度）相等时，新树的秩（高度）加 1。

# 路径压缩

只需在 findfindfind 方法中修改一行代码，就可以让 findfindfind 在查找根节点的同时执行 压缩 的操作。

改动 returnreturnreturn 语句，将递归的 find[parent[x]]find[parent[x]]find[parent[x]] 赋值给 parent[x]parent[x]parent[x] ，让当前查找的节点到根节点路径上所有的节点都指向根节点 。

public int find(int x) {
    if (parent[x] == x) return x;
    return parent[x] = find(parent[x]);
}

在应用带路径压缩的查找和按秩求并后，rank[root]rank[root]rank[root] 记录的值是树实际高度的一个上界，树的实际高度可能小于此值。

# 类的实现

class UnionFind {
    private int[] parent, rank, size; // 按秩求并 和 按大小求并 选择其一

    public UnionFind(int[] parent) {
        this.parent = parent;
        this.rank = new int[parent.length];
        this.size = new int[parent.length];
        Arrays.fill(rank, 1);
        Arrays.fill(size, 1); 
    }

    // 直接查找
    public int findDirectly(int x) {
        if (parent[x] == x) return x;
        return findDirectly(parent[x]);
    }

    // 带路径压缩的查找
    public int find(int x) {
        if (parent[x] == x) return x;
        return parent[x] = find(parent[x]);
    }

    // 直接求并
    public void unionDirectly(int x, int y) {
        int xRoot = find(x), yRoot = find(y);
        if (xRoot != yRoot) {
            parent[yRoot] = xRoot;
        }
    }

    // 按大小求并
    public void unionBySize(int x, int y) {
        int xRoot = find(x), yRoot = find(y);
        if (xRoot != yRoot) { // 根节点不同才求并
            if (size[yRoot] <= size[xRoot]) {
                parent[yRoot] = xRoot;
                size[xRoot] += size[yRoot];
            } else {
                parent[xRoot] = yRoot;
                size[yRoot] += size[xRoot];
            }
        }
    }

    // 按秩求并
    public void union(int x, int y) {
        int xRoot = find(x), yRoot = find(y);
        if (xRoot != yRoot) {
            if (rank[yRoot] <= rank[xRoot]) parent[yRoot] = xRoot;
            else parent[xRoot] = yRoot;
            // 当两棵树秩相等时，新树的秩加 1
            if (rank[xRoot] == rank[yRoot]) rank[xRoot]++;
        }
    }
}

复杂度分析

  • 时间复杂度：带路径压缩的按秩求并的并查集，其查找与合并操作的时间复杂度均为 O(α(n))O(α(n))O(α(n))（α(n)α(n)α(n) 表示增长十分缓慢的反阿克曼函数），对于任何实际的问题的 nnn ，α(n)α(n)α(n) 不会超过 5 。因此可以认为此复杂度为 O(1)O(1)O(1)。初始化 parent[]parent[]parent[]、size[]size[]size[] 或 rank[]rank[]rank[] 数组的时间复杂度为 O(n)O(n)O(n)。
  • 空间复杂度：取决于 parent[]parent[]parent[]、size[]size[]size[] 或 rank[]rank[]rank[] 数组所占空间，为 O(n)O(n)O(n)。
