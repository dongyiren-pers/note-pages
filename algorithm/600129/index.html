# 位运算

# 常见的位运算

# 消除二进制表示中最后一个 1

Brian Kernighan 算法被描述为这样一个结论：记 f(x) 表示 x 和 x − 1 进行 与运算 & 的结果，即 f(x) = x & (x − 1)，那么 f(x) 恰为 x 删去其二进制表示中最右侧的 1 的结果。

x & (x - 1) 的作用是消除数字 x 的二进制表示中的最后一个 1。

::: font-monospace

表达式 二进制表示
x 0011 0100
x - 1 0011 0011
x & (x - 1) 0011 0000

:::

# 保留二进制表示中最后一个 1

x & -x 的作用是保留数字 x 的二进制表示中的最后一个 1。

在计算机中，负数是以补码表示的。负数的补码由其原码取反（除符号位以外）加 1 得到。

::: font-monospace

表达式 二进制表示
x 0011 0100
-x 1100 1100
x & -x 0000 0100

:::

# 异或运算的性质

任何数和 0 做异或运算，结果仍然是原来的数，即 a ^ 0 = a

任何数和其自身做异或运算，结果是 0，即 a ^ a = 0

异或运算满足交换律和结合律，即 a ^ b ^ a = b ^ a ^ a = b ^ (a ^ a) = b ^ 0 = b

# 子集遍历

遍历一个二进制数表示的集合的全部子集，等价于枚举二进制数对应掩码的所有子掩码。

掩码对于源码起到遮罩的作用，掩码中的 111 位意味着源码的相应位得到保留，掩码中的 000 位意味着源码的相应位进行置 000 操作。将掩码的若干 111 位改为 000 位可以得到掩码的子掩码，掩码本身也是自己的子掩码。

给定一个掩码 mmm，希望有效迭代 mmm 的所有子掩码 sss，可以考虑基于位运算技巧的实现。

// 降序遍历 m 的非空子集
int s = m;
while (s > 0) {
    // s 是 m 的一个非空子集
    s = (s - 1) & m;
}

或者使用更紧凑的 for 语句：

// 降序遍历 m 的非空子集
for (int s = m; s > 0; s = (s - 1) & m)
	// s 是 m 的一个非空子集

上述方法不会处理等于 000 的子掩码。

证明

接下来证明，上面的代码访问了所有 mmm 的子掩码，没有重复，并且按降序排列。

假设有一个当前位掩码 sss，并且想继续访问下一个位掩码。在掩码 sss 中减去 111，等价于删除掩码 sss 中最右边的设置位，并将其右边的所有位变为 111。

为了使 s−1s-1s−1 变为新的子掩码，需要删除掩码 mmm 中未包含的所有额外的 111 位，可以使用位运算 (s−1)&m(s-1)\ \&\ m(s−1) & m 来进行此移除。

这两步操作等价于切割掩码 s−1s-1s−1，以确定算术上可以取到的最大值，即按降序排列的 sss 之后的下一个子掩码。因此，该算法按降序生成该掩码的所有子掩码，每次迭代仅执行两个操作。

特殊情况是 s=0s=0s=0。在执行 s−1s-1s−1 之后得到 −1-1−1，其中所有位都为 111。在 (s−1)&m(s-1)\ \&\ m(s−1) & m 操作之后将得到新的 sss 等于 mmm。因此，如果循环不以 s=0s=0s=0 结束，算法的循环将无法终止。

复杂度

使用 popcount(m)\text{popcount}(m)popcount(m) 表示 mmm 二进制中 111 的个数，这种方法可以在 O(2popcount(m))O(2^{\text{popcount}(m)})O(2popcount(m)) 的时间复杂度内遍历集合 mmm 的子集。

# 英文字符大小写转换

通过 或操作 |和 空格 将英文字符转换为小写：

('a' | ' ') = 'a'
('A' | ' ') = 'a'

通过 与操作 & 和 下划线 将英文字符转换为大写：

('b' & '_') = 'B'
('B' & '_') = 'B'

通过 异或操作 ^ 和 空格 进行英文字符大小写互换：

('d' ^ ' ') = 'D'
('D' ^ ' ') = 'd'

以上操作能够产生奇特效果的原因在于这些字符的 ASCII 编码值通过位运算恰巧就能得到正确的结果，有兴趣的读者可以查 ASCII 编码表进行验证。

# 判断两个数是否异号

int x = -1, y = 2;
boolean f = ((x ^ y) < 0); // true

int x = 3, y = 2;
boolean f = ((x ^ y) < 0); // false

这个技巧还是很实用的，利用的是补码编码的符号位。如果不用位运算来判断是否异号，需要使用 if-else 分支。你可能想利用乘积或者商来判断两个数是否异号，但是这种处理方式可能造成溢出。

# 不使用临时变量交换两个数

int a = 1, b = 2;
a ^= b; 
b ^= a; 
a ^= b;
// 现在 a = 2, b = 1

# 加一和减一

int n = 1;
n = -~n;
// 现在 n = 2

int n = 2;
n = ~-n;
// 现在 n = 1
