# 回文子串预处理

给定长度为 nnn 的字符串 sss。对于任意索引对 (i,j)(i,j)(i,j)，判断子串 s[i...j]s[i...j]s[i...j] 是否为回文串。

核心思想

当我们在判断 s[i...j]s[i...j]s[i...j] 是否为回文串时，常规的方法是使用双指针分别指向 iii 和 jjj，每次判断两个指针指向的字符是否相同，直到两个指针相遇。然而这种方法会产生重复计算。

因此，我们可以将字符串 sss 的每个子串 s[i...j]s[i...j]s[i...j] 是否为回文串预处理出来，使用动态规划即可。设 f(i,j)f(i, j)f(i,j) 表示 s[i..j]s[i..j]s[i..j] 是否为回文串，那么有状态转移方程：

统一奇偶回文串

偶回文子串可以通过类似的处理得到，这里介绍一种方式可以将奇偶回文串巧妙地统一起来。

奇偶回文串唯一的区别在于，奇回文串的中心是一个字符，而偶回文串的中心在两个字符之间，这让我们代码处理起来不够统一。

我们可以在所有字符间插入特殊字符，使得偶回文串的中心也是字符，例如：abcabc，在其中插入特殊符号 #，那么它就变成了 #a#b#c#a#b#c#。这样，奇回文串的中心是原本的字符，偶回文串的中心就是我们插入的 # 字符了，至此奇偶回文串的处理代码便巧妙地统一了。

我们在回文串的首位再增加两个肯定不会匹配的字符 $ 与 ^，使其变成 $#a#b#c#a#b#c#^，这样我们连越界判断都可以忽略了，因为到了边界一定不能匹配。

# 最长回文子串

给定一个字符串 sss，找到 sss 中最长的回文子串。

回文子串预处理

public String longestPalindrome(String s) {
    int n = s.length();
    boolean[][] f = new boolean[n][n];
    for (int i = 0; i < n; i++)
        Arrays.fill(f[i], true);
    int start = 0, end = 0;
    for (int i = n - 1; i >= 0; i--) {
        for (int j = i + 1; j < n; j++) {
            f[i][j] = s.charAt(i) == s.charAt(j) && f[i + 1][j - 1];
            if (f[i][j] && j - i > end - start) {
                start = i;
                end = j;
            }
        }
    }
    return s.substring(start, end + 1);
}

// 统一奇偶回文串
public String handle(String s) {
    StringBuilder sb = new StringBuilder();
    sb.append("$");
    for (int i = 0; i < s.length(); i++)
        sb.append("#").append(s.charAt(i));
    sb.append("#");
    sb.append("^");
    return sb.toString();
}

// Manacher 算法
public int[] manacher(String s) {
    int n = s.length();
    int[] p = new int[n];
    int l = 1, r = 1;
    for (int i = 1; i < n - 1; i++) {
        p[i] = Math.max(0, Math.min(r - i, p[l + r - i]));
        while (s.charAt(i - p[i] - 1) == s.charAt(i + p[i] + 1))
            p[i]++;
        if (i + p[i] > r) {
            l = i - p[i];
            r = i + p[i];
        }
    }
    return p;
}

public String longestPalindrome(String s) {
    int[] p = manacher(handle(s));
    int maxIdx = 1;
    for (int i = 1; i < p.length - 1; i++) {
        if (p[i] > p[maxIdx])
            maxIdx = i;
    }
    int start = (maxIdx - p[maxIdx] - 1) / 2;
    return s.substring(start, start + p[maxIdx]);
}

# 寻找子串位置

给定一个文本 sss 和一个字符串 ppp，我们尝试找到并展示 ppp 在 sss 中 首次出现的位置。nnn 表示文本 sss 的长度，mmm 表示字符串 ppp 的长度。

我们构造一个字符串 p+#+sp + \# + sp+#+s，其中 #\## 为一个既不出现在 ppp 中也不出现在 sss 中的分隔符。接下来计算该字符串的前缀函数。由于分隔符的存在，该长度不可能超过 mmm。而如果等式 π[i]=m\pi[i] = mπ[i]=m 成立，则意味着 ppp 完整出现在该位置（即其右端点位于位置 iii）。因此，如果在某一位置 iii 有 π[i]=m\pi[i] = mπ[i]=m 成立，则字符串 ppp 在字符串 sss 的 i−m+1−(m+1)=i−2mi - m + 1 - (m + 1) = i - 2mi−m+1−(m+1)=i−2m 处出现。

前缀函数的值永远不超过一特定值，因此，我们不需要存储整个字符串以及整个前缀函数，而只需要二者开头的一部分。在这种情况下意味着只需要存储字符串 p+#p + \#p+# 以及相应的前缀函数值即可。我们可以一次读入字符串 sss 的一个字符并计算当前位置的前缀函数值。

  • 无需显式地创建字符串 p+#+sp + \# + sp+#+s。
  • 无需显式地保存所有前缀函数的结果，而只需要保存字符串 ppp 部分的前缀函数即可。

# 形式化定义

一个 确定有限状态自动机（DFA） 由以下五部分构成：

  • 字符集（Σ\SigmaΣ）。该自动机只能输入这些字符。
  • 状态集合（QQQ）。如果把 DFA 看成有向图，那么 DFA 中的状态就相当于图上的顶点。
  • 起始状态（startstartstart）。start∈Qstart\in Qstart∈Q，是一个特殊的状态。起始状态一般用 sss 表示，为了避免混淆，本文中使用 startstartstart。
  • 接受状态集合（FFF）。F⊆QF\subseteq QF⊆Q，是一组特殊的状态。
  • 转移函数（δ\deltaδ）。δ\deltaδ 是一个接受两个参数返回一个值的函数，其中第一个参数和返回值都是一个状态，第二个参数是字符集中的一个字符。如果把 DFA 看成有向图，那么 DFA 中的转移函数就相当于顶点间的边，而每条边上都有一个字符。

DFA 的作用就是识别字符串，一个自动机 AAA，若它能识别（接受）字符串 SSS，那么 A(S)=TrueA(S)=\mathrm{True}A(S)=True，否则 A(S)=FalseA(S)=\mathrm{False}A(S)=False。

当一个 DFA 读入一个字符串时，从初始状态起按照转移函数一个一个字符地转移。如果读完一个字符串后位于一个接受状态，那么称这个 DFA 接受 这个字符串，反之称这个 DFA 不接受 这个字符串。

如果一个状态 vvv 没有字符 ccc 的转移，那么我们令 δ(v,c)=null\delta(v,c)=\mathrm{null}δ(v,c)=null，而 null\mathrm{null}null 只能转移到 null\mathrm{null}null，且 null\mathrm{null}null 不属于接受状态集合。无法转移到任何一个接受状态的状态都可以视作 null\mathrm{null}null，或者说，null\mathrm{null}null 代指所有无法转移到任何一个接受状态的状态。

我们扩展定义转移函数 δ\deltaδ，令其第二个参数可以接收一个字符串：δ(v,s)=δ(δ(v,s[1]),s[2..∣s∣])\delta(v,s)=\delta(\delta(v,s[1]),s[2..|s|])δ(v,s)=δ(δ(v,s[1]),s[2..∣s∣])，扩展后的转移函数就可以表示从一个状态起接收一个字符串后转移到的状态。那么，A(s)=[δ(start,s)∈F]A(s)=[\delta(start,s)\in F]A(s)=[δ(start,s)∈F]。

# 有效数字

有效数字（按顺序）可以分成以下几个部分：

  • 一个 小数 或者 整数
  • （可选）一个 'e' 或 'E' ，后面跟着一个 整数

小数（按顺序）可以分成以下几个部分：

  • （可选）一个符号字符（'+' 或 '-'）
  • 下述格式之一：
    • 至少一位数字，后面跟着一个点 '.'
    • 至少一位数字，后面跟着一个点 '.' ，后面再跟着至少一位数字
    • 一个点 '.' ，后面跟着至少一位数字

整数（按顺序）可以分成以下几个部分：

  • （可选）一个符号字符（'+' 或 '-'）
  • 至少一位数字

部分有效数字列举如下：["2", "0089", "-0.1", "+3.14", "4.", "-.9", "2e10", "-90E3", "3e+7", "+6e-1", "53.5e93", "-123.456e789"]

部分无效数字列举如下：["abc", "1a", "1e", "e3", "99e2.5", "--6", "-+3", "95a54e53"]

给定一个字符串 s ，如果 s 是一个 有效数字 ，请返回 true 。

enum State {
    INITIAL, INT_SIGN, INT, POINT, POINT_WITHOUT_INT, FRACTION, EXP, EXP_SIGN, EXP_INT, END
}

enum CharType {
    SIGN, INT, POINT, EXP, ILLEGAL
}

public CharType toChartType(char c) {
    if (c == '+' || c == '-')
        return CharType.SIGN;
    else if ('0' <= c && c <= '9')
        return CharType.INT;
    else if (c == '.')
        return CharType.POINT;
    else if (c == 'e' || c == 'E')
        return CharType.EXP;
    else return CharType.ILLEGAL;
}

public boolean isNumber(String s) {
    Map<State, Map<CharType, State>> transfer = new HashMap<>();
    Map<CharType, State> stateInitial = new HashMap<CharType, State>() {{
        put(CharType.SIGN, State.INT_SIGN);
        put(CharType.INT, State.INT);
        put(CharType.POINT, State.POINT_WITHOUT_INT);
    }};
    transfer.put(State.INITIAL, stateInitial);
    Map<CharType, State> stateIntSign = new HashMap<CharType, State>() {{
        put(CharType.INT, State.INT);
        put(CharType.POINT, State.POINT_WITHOUT_INT);
    }};
    transfer.put(State.INT_SIGN, stateIntSign);
    Map<CharType, State> stateInt = new HashMap<CharType, State>() {{
        put(CharType.INT, State.INT);
        put(CharType.POINT, State.POINT);
        put(CharType.EXP, State.EXP);
    }};
    transfer.put(State.INT, stateInt);
    Map<CharType, State> statePoint = new HashMap<CharType, State>() {{
        put(CharType.EXP, State.EXP);
        put(CharType.INT, State.FRACTION);
    }};
    transfer.put(State.POINT, statePoint);
    Map<CharType, State> statePointWithoutInt = new HashMap<CharType, State>() {{
        put(CharType.INT, State.FRACTION);
    }};
    transfer.put(State.POINT_WITHOUT_INT, statePointWithoutInt);
    Map<CharType, State> stateFraction = new HashMap<CharType, State>() {{
        put(CharType.INT, State.FRACTION);
        put(CharType.EXP, State.EXP);
    }};
    transfer.put(State.FRACTION, stateFraction);
    Map<CharType, State> stateExp = new HashMap<CharType, State>() {{
        put(CharType.SIGN, State.EXP_SIGN);
        put(CharType.INT, State.EXP_INT);
    }};
    transfer.put(State.EXP, stateExp);
    Map<CharType, State> stateExpSign = new HashMap<CharType, State>() {{
        put(CharType.INT, State.EXP_INT);
    }};
    transfer.put(State.EXP_SIGN, stateExpSign);
    Map<CharType, State> stateExpInt = new HashMap<CharType, State>() {{
        put(CharType.INT, State.EXP_INT);
    }};
    transfer.put(State.EXP_INT, stateExpInt);

    int n = s.length();
    State state = State.INITIAL;
    for (int i = 0; i < n; i++) {
        CharType charType = toChartType(s.charAt(i));
        if (!transfer.get(state).containsKey(charType)) {
            return false;
        } else {
            state = transfer.get(state).get(charType);
        }
    }
    return state == State.INT || state == State.POINT || state == State.FRACTION || state == State.EXP_INT;
}
