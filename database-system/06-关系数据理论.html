# 第6章 关系数据理论

# 6.1 问题的提出

作为二维表，关系要符合一个最基本的条件：每个分量必须是不可分开的数据项。满足了这个条件的关系模式就属于第一范式（1NF）。

数据依赖：是一个关系内部 属性与属性 之间的一种 约束关系。

一个 “好” 的模式 应当不会发生 插入异常、删除异常 和 更新异常，数据冗余 应尽可能少。

# 6.2 规范化

# 函数依赖

设 R(U)R(U)R(U) 是一个关系模式，UUU 是 RRR 的属性集合，XXX 和 YYY 是 UUU 的子集。对于 R(U)R(U)R(U) 的任意一个可能的关系 rrr，如果 rrr 中不存在两个元组，它们在 XXX 上的属性值相同，而在 YYY 上的属性值不同，则称 “ XXX 函数确定 YYY ” 或 “ YYY 函数依赖于 XXX ”，记作 X→YX\rightarrow YX→Y。

# 平凡函数依赖与非平凡函数依赖

X→YX→YX→Y，但 Y⊆XY⊆XY⊆X 则称 X→YX→YX→Y 是 平凡函数依赖。

X→YX→YX→Y，但 Y⊈XY⊈XY⊈X 则称 X→YX→YX→Y 是 非平凡函数依赖。

对于任一关系模式，平凡函数依赖都是必然成立的，它不反映新的语义。若不特别声明， 我们总是讨论非平凡函数依赖。

# 完全函数依赖与部分函数依赖

在 R(U)R(U)R(U) 中，如果 X→YX\rightarrow YX→Y，并且对于 XXX 的任何一个真子集 X′X^\primeX′，都有 X′↛YX^\prime\nrightarrow YX′↛Y，则称 YYY 对 XXX 完全函数依赖，记作：X\buildrelF→YX\buildrel{F}\over \rightarrow Y→YX\buildrelF​。

若 X→YX\rightarrow YX→Y，但 YYY 不完全函数依赖于 XXX，则称 YYY 对 XXX 部分函数依赖，记作：X\buildrelP→YX\buildrel{P}\over \rightarrow Y→YX\buildrelP​。

# 传递函数依赖

在 R(U)R(U)R(U) 中，如果 X→Y,Y⊈X,Y↛X,Y→Z,Z⊈YX\rightarrow Y,\ \ Y\nsubseteq X,\ \ Y\nrightarrow X,\ \ Y\rightarrow Z,\ \ Z\nsubseteq YX→Y,  Y⊈X,  Y↛X,  Y→Z,  Z⊈Y，则称 ZZZ 对 XXX 传递函数依赖，记作：X\buildrel传递→YX\buildrel{\text{传递}}\over \rightarrow Y→YX\buildrel传递​。

# 码

候选码：设 KKK 为 R<U,F>R<U,F>R<U,F> 中的属性或属性组合。若 K\buildrelF→UK\buildrel{F}\over \rightarrow U→UK\buildrelF​，则 KKK 称为 RRR 的一个候选码。

超码：如果 K\buildrelP→UK\buildrel{P}\over \rightarrow U→UK\buildrelP​，则 KKK 称为超码。候选码是最小的超码，即 KKK 的任意一个真子集都不是候选码。

主码：若关系模式 RRR 有多个候选码，则选定其中的一个做为主码。

主属性与非主属性：包含在任何一个候选码中的属性 ，称为主属性；不包含在任何码中的属性称为非主属性。

全码：整个属性组是码，称为全码。

外码：关系模式 RRR 中属性或属性组 XXX 并非 RRR 的码，但 XXX 是另一个关系模式的码，则称 XXX 是 RRR 的外码。

# 范式

范式是符合某一种级别的关系模式的集合。

关系数据库中的关系必须满足一定的要求。满足不同程度要求的为不同范式。

某一关系模式 RRR 为第 nnn 范式，可简记为 R∈nNFR∈nNFR∈nNF。

# 1NF

如果一个关系模式 RRR 的所有属性都是不可分的基本数据项，则 R∈1NFR\in1NFR∈1NF。

# 2NF

若关系模式 R∈1NFR\in1NFR∈1NF，并且每一个非主属性都完全函数依赖于 RRR 的码，则 R∈2NFR\in2NFR∈2NF。

消除非主属性对码的部分函数依赖。

# 3NF

关系模式 R<U,F>R<U, F>R<U,F> 中若不存在这样的码 XXX，属性组 YYY 及非主属性 Z(Z⊈Y)Z(Z\nsubseteq Y)Z(Z⊈Y) 使得 $ X\rightarrow Y，，，(Y\nrightarrow X)，，，Y\rightarrow Z$ 成立，则称 R<U,F>∈3NFR<U,F>\in3NFR<U,F>∈3NF。

消除非主属性对码的传递函数依赖。

# BCNF

关系模式 R<U,F>∈1NFR<U,F>\in1NFR<U,F>∈1NF，若 X→YX\rightarrow YX→Y 且 Y⊈XY\nsubseteq XY⊈X 时 XXX 必含有码，R<U,F>∈BCNFR<U,F>\in BCNFR<U,F>∈BCNF。

消除主属性对码的部分和传递函数依赖。

# 4NF

关系模式 R<U,F>∈1NFR<U, F>\in1NFR<U,F>∈1NF，如果对于 RRR 的每个非平凡多值依赖 X→→Y(Y⊈X)X\rightarrow\rightarrow Y\ (Y\nsubseteq X)X→→Y (Y⊈X)，XXX 都含有码，则称 R<U,F>∈4NFR<U,F>\in4NFR<U,F>∈4NF。

消除非平凡且非函数依赖的多值依赖。

多值依赖

设 R(U)R(U)R(U) 是属性集 UUU 上的一个关系模式，XXX、YYY、ZZZ 是 UUU 的子集，并且 Z=U−X−YZ=U-X-YZ=U−X−Y。关系模式 R(U)R(U)R(U) 中多值依赖 X→→YX\rightarrow\rightarrow YX→→Y 成立，当且仅当对 R(U)R(U)R(U) 的任一关系 rrr，给定的一对 (x,z)(x,z)(x,z) 值，有一组 YYY 的值，这组值仅仅决定于 xxx 值而与 zzz 值无关。

# 6.3 数据依赖的公理系统

# Armstrong 公理系统

Armstrong 公理系统：一套推理规则，是模式分解算法的理论基础。

设 UUU 为属性集总体，FFF 是 UUU 上的一组函数依赖， 于是有关系模式 R<U,F>R <U,F >R<U,F>。对 R<U,F>R <U,F>R<U,F> 来说有以下的推理规则：

  • 自反律：若 Y⊆X⊆UY \subseteq X \subseteq UY⊆X⊆U，则 X→YX →YX→Y 为 FFF 所蕴涵。
  • 增广律：若 X→YX→YX→Y 为 FFF 所蕴涵，且 Z⊆UZ\subseteq UZ⊆U，则 XZ→YZXZ→YZXZ→YZ 为 FFF 所蕴涵。
  • 传递律：若 X→YX→YX→Y 及 Y→ZY→ZY→Z 为 FFF 所蕴涵，则 X→ZX→ZX→Z 为 FFF 所蕴涵。

根据这三条推理规则可以得到下面三条推理规则：

  • 合并规则：由 X→YX→YX→Y，X→ZX→ZX→Z，有 X→YZX→YZX→YZ。
  • 伪传递规则：由 X→YX→YX→Y，WY→ZWY→ZWY→Z，有 XW→ZXW→ZXW→Z。
  • 分解规则：由 X→YX→YX→Y 及 Z⊆YZ\subseteq YZ⊆Y，有 X→ZX→ZX→Z。
